# SQL Analysis

## Overview

This folder contains SQL queries used to analyze the Customer Churn dataset. The analysis helps identify customer churn patterns, customer behavior, and business trends using SQL.

## Objective

The objective of this analysis is to extract meaningful business insights from the Customer Churn dataset through SQL queries.

## Database

- MySQL
- Customer Churn Database

## Dataset

- Customer Churn Dataset
- Total Records: 7,043

## SQL Concepts Used

- SELECT
- WHERE
- ORDER BY
- GROUP BY
- HAVING
- COUNT()
- SUM()
- AVG()
- ROUND()
- Aggregate Functions

## Analysis Performed

- Total Customers
- Churn Customers
- Active Customers
- Churn Rate
- Gender-wise Analysis
- Contract-wise Analysis
- Payment Method Analysis
- Monthly Charges Analysis
- Total Charges Analysis
- Tenure Analysis

## Key Insights

- Month-to-Month contract customers have the highest churn.
- Electronic Check users show a higher churn rate.
- Customers with higher monthly charges are more likely to churn.
- Longer-tenure customers are less likely to churn.

## Output

The SQL analysis provides business insights that support customer retention strategies and improve decision-making through data-driven analysis.
